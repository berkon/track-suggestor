Track Suggestor

General

Track Suggestor is a tool for Native Instrument’s “Traktor Pro” which suggests tracks for harmonic mixing based on the last track that was loaded in one of Traktor’s decks. It does this based on certain criteria. First a track must roughly match the BPM rate. Then also the key must match. Matching in this case means that the following “harmonic mixing” rules apply to a track:

  • Key is 1 step higher/lower
  • Key is identical but has different major/minor type

Track Suggestor shows the matching tracks in the track suggestion window and also creates two playlists, which can be used directly from within Traktor:

  • Recommended.nml

This playlist has the same content like the track suggestion window. That means tracks which match the harmonic mixing rules.

  • Recommended_7.nml

This is a specific playlist which contains tracks who’s key is 7 steps higher/lower, which means

exactly one halftone up/down. In order to mix these tracks, the “Key” knob in Traktor must be

turned exactly 1.00 unit up/down accordingly, which correlates as well to 1 halftone up/down. To

get integer values (which is necessary) I recommend to use the little +/- buttons on the knob.

When turning it with the mouse, it is very hard to get an integer value. Now that track should

match well with the currently running track.

Connection

Track Suggestor connects to Traktor via MIDI. A specific MIDI controller, which is capable of sending track and artist names via MIDI must be configured in Traktor’s Controller Manager. Currently only the Denon DH-HC4500 supports this. Furthermore a virtual MIDI connector must be present on the system. On Apple devices such virtual MIDI ports can be created with onboard tools. On Windows, third party tools like loopMIDI or CoperLAN can be used.

First open Traktor, go to Settings and open the Controller Manager. Add a Denon ⇒  DN-HC4500 device and set the Out-Port to the virtual MIDI port which you’ve configured on your system before. In this case loopMIDI is used on Windows

Now that the MIDI connection is set up, please restart Track Suggester once in order to let it rescan the MIDI ports. It will automatically show the settings page at startup as long as an issue with one of the following settings is found. So please make sure to set the following settings properly:

  • Virtual MIDI port to which Traktor was just connected.

  • Location of the collection file (collection.nml) which Traktor uses. Don’t worry, it is used read only! Nothing will be modified!

  • Select a directory where the generated playlists shall be stored to. They can then be selected via the “Explorer” folder from within Traktor. E.g. you can put them in the Demo Content folder like in this example:

This way you can reach them very quickly without having to click through a lot of folders. On Windows the location of this folder is in: C:\ProgramData\Native Instruments\Traktor 2\Factory Sounds.

IMPORTANT

The real DN-HC4500 device is not capable of handling text. It just displays characters which have been written to certain display addresses/elements of the display. In order to display long names, Traktor simulates scrolling. It does this by shifting names one character at a time to the left and sends this new string over and over again to the controller. In order to obtain a reasonable scrolling effect, characters must not be sent too fast (approx 2 character/s). As Track Suggestor utilizes this mechanism, it has to wait until all characters have been scrolled at least once through the display. This means that extremely long names can sometimes take 10 (or more) seconds to load!

IMPORTANT

Traktor isn’t prepared to handle external playlists which change dynamically. So when a new track is loaded into a Traktor deck and Track Suggestor in turn creates new playlists based on this new track, the new content of the playlist does not show up in Traktor’s browser window! A little silly workaround must be applied to see the changes: Click on a different playlist and then select the initial one again.
