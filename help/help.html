Track Suggestor

  1. General

Track Suggestor is a tool for Native Instrument’s “Traktor Pro” which suggests tracks for harmonic mixing based on the last track that has been loaded in one of Traktor’s decks. It does this based on certain criteria. First a track must roughly match the BPM rate. Then also the key must match. Matching in this case means that the following rules apply to a track:

  • Key is 1 step higher/lower
  • Key is identical but has different major/minor type

Except for the track list shown in the track suggestion window, Track Suggestor also creates two playlists, which Traktor can read:

Recommended.nml

This playlist contains tracks that match the harmonic mixing rules described above. It has the same content like the track list shown in the tool.

Recommended_7.nml

This playlist contains tracks where the key is 7 steps higher/lower, which means one halftone

up/down. In order to mix these tracks, the “Key” knob in Traktor must be turned 1 unit up or

down, which correlates to 1 halftone up/down. Thus the pitch of that track is increased decreased

by 1 halftone and will now match the track which was used as a base for the calculation (and is

most likely currently running).

  1. Connection

Track Suggestor connects to Traktor via MIDI. A specific MIDI controller, which is capable of sending track and artist names via MIDI must be configured in Traktor’s Controller Manager. Currently only the Denon DH-HC4500 supports this. Furthermore a virtual MIDI connector must be present on the system. On Apple devices such virtual MIDI ports can be created with onboard tools. On Windows, third party tools like loopMIDI or CoperLAN can be used.

First open Traktor, go to Settings and open the Controller Manager. Add a Denon ⇒  DN-HC4500 device. Now set the Out-Port to the virtual MIDI port which you’ve configured on your system. In this case loopMIDI is used on Windows

This means that Traktor will now send track and artist names to this virtual port. Now that the MIDI connection is set up, the tool can be started. It will automatically show the settings page at startup as long as setup issues are found. Please configure the following settings:

  • Select the virtual MIDI port to which Traktor was just connected.

  • Select the collection file (collection.nml) which Traktor uses. Don’t worry, it is read only! Nothing will be modified!

  • Select a directory where the playlists containing the suggestions shall be stored to. They can then be selected via the “Explorer” folder from within Traktor. E.g. you can put it in the Demo Content folder like in this example:

This way you can reach them very quickly without having to click through a lot of folders. On Windows the demo content is in: C:\ProgramData\Native Instruments\Traktor 2\Factory Sounds.

IMPORTANT

Traktor does not send track and artist names at once to the DN-HC4500 MIDI controller, because that hardware device is not capable of receiving that data at once. Instead each character is sent one by one and the device is shifting all received characters leftwards in the display as soon as it receives a new one. This creates the impression that the names are rolling through the display. To obtain this effect characters must not be sent too fast (approx 1 character per 0,5s). As Track Suggestor utilizes this mechanism, it has to wait until all characters have been sent at least once. This means that long names can sometimes take 10 (or more) seconds to load in Track Suggestor!

IMPORTANT

Traktor isn’t prepared to handle external playlists that change dynamically. So when a new track is loaded into a Traktor deck and Track Suggestor in turn creates new playlists based on this new track, the new content does not show up in Traktor’s browser window, as someone would expect! A workaround must be performed in order to see the changes. First another playlist must be selected (clicked), and then the initial one must be selected again.
